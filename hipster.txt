1.Introduction:
------------------
#GIT:
a.Distributed Source Control System.
    .Not required to be decentralized.
b.Massively scales
c.Open source
d.Developed for Linux project requirements
e.Most operations are local
f.Very fast
g.Active Community
h.Most popular DVCS, VCS
	
=>Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

#Key Concepts:
a.Repository contains files, history, config managed by Git.
b.Three states of GIT:
.Working directory
.Staging area - pre commit holding area
.Commit - Git repository(history) 

c.Remote Repository(GitHub)
d.Master branch

3.Git Quick Start:
--------------------
1.Login to Github account and create github-demo repository.
2.Create folder with name project into your local machine and right click inside folder and click-on Git bash here command prompt.
3.Run below command if you have not yet configured.

$ git config --global user.name "Amarkant Gupta"

$ git config --global user.email "amarkantgupta1@gmail.com"

$ git config --global --list

You can see below results:

$ git config --global --list
user.email=amarkantgupta1@gmail.com
user.name=Amarkant Gupta

4.Now, we will clone github-demo repository into our local machine.

$ git clone https://github.com/amarkantgupta1/github-demo.git

5.Now, enter ls command to show us current directory.

$ ls

6.Go to repository.

$ cd github-demo/

7.Check the branch repository status using below cmd and it will show us default branch as master branch because we have not created any branch yet.

$ git status

$ git status
On branch master
Your branch is up to date with 'origin/master'.

nothing to commit, working tree clean

8.Now, We will create one text file with some text value using below command and then press enter and type ls cmd to see file is created or not?

$ echo "Test Git Quick Start Demo" >>start.txt

9. Using cat command we can see or display text file content on cmd prompt.

$ cat start.txt
Test Git Quick Start Demo

10.After entering below cmd show as there is untracked files with name start.txt and use "git add" cmd to include and then commit file.
$ git status
On branch master
Your branch is up to date with 'origin/master'.

Untracked files:
  (use "git add <file>..." to include in what will be committed)

        start.txt

nothing added to commit but untracked files present (use "git add" to track)

11. After adding file check status now and it will show and highlights file name into green color( new file:   start.txt).
12. And this file will moved from "Working Directory" area to Local "Staging Area".
